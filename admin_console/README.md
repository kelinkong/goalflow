# GoalFlow Admin Console

GoalFlow 管理台前端。

它不是独立产品，而是 GoalFlow 成长系统的后台管理入口。  
管理台需要和用户端保持同一套产品口径：

1. `目标`
用户想做成什么事

2. `习惯`
用户想成为哪一种人

3. `复盘`
用户如何理解自己的生活

管理台的职责不是复刻用户端页面，而是帮助管理员观察、审核、排查和维护这三层系统相关数据。

## 当前定位

当前管理台仍处于开发阶段，已实现的能力主要集中在：

1. 管理员登录
2. 用户列表查看
3. 仪表盘统计
4. 模板列表与模板创建
5. 后端日志查看

这意味着：

1. 当前管理台对 `目标` 已有基础观察能力
2. 对 `模板` 已有一定管理能力
3. 对 `习惯` 和 `复盘` 暂未形成完整后台模块

后续后台能力需要按新的三层模型逐步补齐，而不是继续围绕旧的“目标管理后台”概念扩展。

## 与用户端的一致口径

用户端的正式方案文档：

1. [growth-system-requirements.md](./docs/growth-system-requirements.md)

## 当前已实现页面

当前路由与页面如下：

1. `Login.vue`
管理员登录页

2. `Dashboard.vue`
仪表盘

3. `Users.vue`
用户列表

4. `Templates.vue`
模板列表 / 审核相关视图

5. `TemplateCreate.vue`
模板创建页

6. `Logs.vue`
后端日志查看

## 当前管理台负责什么

### 1. 系统观察

1. 查看用户数量
2. 查看目标数量
3. 查看基础任务统计

### 2. 模板管理

1. 创建模板
2. 查看模板
3. 审核模板

### 3. 用户排查

1. 查看用户列表
2. 辅助定位用户数据问题

### 4. 日志排查

1. 查看后端日志
2. 辅助定位接口异常

## 后续应补齐的后台模块

围绕新的产品三层模型，管理台后续建议补齐以下模块。

### 1. 目标管理

目标：

1. 查看目标列表
2. 查看目标状态
3. 查看目标完成情况
4. 辅助排查目标任务数据异常

### 2. 习惯管理

目标：

1. 查看习惯定义
2. 查看习惯打卡记录
3. 查看习惯完成趋势
4. 辅助排查习惯连续记录异常

说明：

这是新的后台模块，目前尚未实现。

### 3. 复盘管理

目标：

1. 查看每日复盘记录数量
2. 查看复盘维度完成情况
3. 辅助排查复盘接口和数据问题

说明：

复盘不需要做成复杂审核系统，但需要有基础可观测性。

### 4. 轨迹观察

目标：

1. 按日期聚合查看目标、习惯、复盘
2. 辅助定位“某天为什么展示不对”的问题

说明：

轨迹是展示层概念，后台未必需要独立表，但需要有排查入口。

## 安装依赖

```bash
cd ./admin_console
npm install
```

## 启动开发环境

```bash
cd ./admin_console
npm run dev
```

默认访问地址：

```text
http://localhost:3000
```

## 默认管理员账户

```text
邮箱：admin@goalflow.com
密码：admin123
```

## API 代理

开发环境通过 `vite.config.js` 代理 `/api` 到本地后端。

默认要求本地后端已启动：

```text
http://127.0.0.1:8081
```

## 构建

```bash
cd ./admin_console
npm run build
```

## 文档维护原则

后续如果管理台新增模块，文档必须同步回答两个问题：

1. 这个模块服务于目标、习惯、复盘中的哪一层？
2. 这个模块是在“管理数据”、还是在“排查问题”、还是在“审核内容”？

如果回答不清楚，就说明模块设计边界还不够清晰。
