# GoalFlow 1.0.1 打磨计划

**项目名称**：GoalFlow  
**目标版本**：1.0.1  
**文档日期**：2026-03-31  
**文档类型**：版本打磨说明 + 实施顺序

## 一、版本定位

1.0.1 不新增复杂功能，只打磨现有体验。

这个版本的目标不是让用户做得更多，而是让现有页面更顺手、更温和、更耐看，让用户在打开 app 时不会被信息和语气压住。

## 二、版本原则

1. 不增加新的状态系统、模式切换或额外流程
2. 优先优化现有页面的信息密度、交互负担和视觉噪音
3. 文案继续保持陪伴式表达，不回到督促式语气
4. 每次修改都应让页面更直观，而不是更花哨
5. 优先做小而完整的收口，不做半成品功能

## 三、版本范围

### 3.1 本版本要做

1. 首页信息层级和卡片节奏打磨
2. 轨迹页趋势图、日历和提示语继续收口
3. 复盘页输入负担和视觉层级优化
4. 全局 toast、空状态、提示语统一
5. 现有页面的细节问题修复

### 3.2 本版本不做

1. 低能量模式
2. 新的每日状态系统
3. 轻量复盘新结构
4. 本周观察模块
5. AI 建议、情绪分析、复杂提醒策略

## 四、打磨方向

## 4.1 首页

目标：打开后更轻、更清楚，不让人一上来就感到任务压力。

优化重点：

1. 强化“今天是什么”与“现在可以做什么”的层级
2. 减少卡片之间的互相抢注意力
3. 继续收文案，避免“待完成”带来的压迫感
4. 空状态和无数据时保持温和，不显得冷或空

验收标准：

1. 用户 3 秒内能看懂首页主要信息
2. 首页不会出现明显催促式语气
3. 卡片主次关系清楚，没有视觉堆叠感

## 4.2 轨迹页

目标：让轨迹页更像“回看变化”，而不是“接受考核”。

优化重点：

1. 趋势图继续保持简单直观，避免装饰性过强
2. 日历状态表达清楚，不制造误读
3. 提示语更中性、更像观察
4. 页面整体减少视觉噪音，提升留白和节奏

验收标准：

1. 趋势图一眼能看懂，不需要解释
2. 日历状态不出现歧义
3. 提示语描述事实，不制造焦虑

## 4.3 复盘页

目标：让用户更容易开始写，而不是一看到页面就想退出。

优化重点：

1. 降低输入前的心理门槛
2. 优化表单间距、标题层级和说明文案
3. 让“写一点”也显得自然，不必像在做作业
4. 减少不必要的校验压迫感

验收标准：

1. 页面结构更清楚，输入起点更明显
2. 文案不会让用户感觉必须写很多
3. 保存反馈简洁、温和、明确

## 4.4 全局文案与反馈

目标：让全 app 的语气一致，不同页面不要像不同产品。

优化重点：

1. 统一 toast 的语气和长度
2. 统一空状态文案
3. 统一提醒、成功、失败提示的表达方式
4. 去掉过度用力、带评判感或传播焦虑的句子

验收标准：

1. 相似场景下的提示语风格一致
2. 不出现明显“命令式”或“打鸡血”表达
3. 出错提示优先告诉用户发生了什么，而不是责备用户

## 五、实施顺序

### 第一阶段：文案与页面收口

1. 首页高频提示语收一轮
2. 轨迹页提示语和说明文案收一轮
3. 复盘页提示语和保存反馈收一轮

### 第二阶段：视觉与交互打磨

1. 首页卡片层级和间距优化
2. 轨迹页图表和日历样式优化
3. 复盘页表单视觉层级优化

### 第三阶段：统一反馈体验

1. 统一 toast 文案
2. 统一空状态表达
3. 修复现有页面中的突兀细节

## 六、开发约束

1. 不新增数据库表
2. 不新增新的业务流程
3. 不新增新的主入口
4. 如需补文案规范，优先更新 `docs/content-tone-guide.md`
5. 如需调整产品方向，先更新本文档再改代码

## 七、文档同步要求

本版本开发时至少同步维护：

1. 本文档
2. `docs/content-tone-guide.md`
3. 如页面定位有明显变化，补充对应页面说明

## 八、版本结论

1.0.1 的重点不是加法，而是减法。

它要把 GoalFlow 从“功能继续变多”收回来，变成一个更克制、更顺手、更有人情味的记录工具。
